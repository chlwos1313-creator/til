# 자료 구조 실습
## Dictionary (딕셔너리) 주요 메서드
데이터를 키(Key)-값(Value) 쌍으로 관리하며, 매우 빠른 탐색 속도를 자랑하는 자료구조입니다.

-get(key[, default]): 키를 조회하며, 키가 없을 때 KeyError를 내는 대신 None이나 설정한 기본값을 반환합니다.

-.keys() / .values() / .items(): 딕셔너리의 키 목록, 값 목록, 혹은 (키, 값) 쌍을 뷰(View) 객체 형태로 반환합니다.

-.pop(key[, default]): 특정 키의 항목을 제거하고 그 값을 반환합니다.

-.update(other): 다른 딕셔너리의 데이터를 합치거나 기존 값을 수정합니다.

-.setdefault(key, default): 키가 없으면 기본값을 저장한 뒤 반환하고, 있으면 기존 값을 그대로 반환합니다.

-.clear(): 딕셔너리 내부의 모든 데이터를 삭제합니다.

## Set (세트) 주요 메서드
중복을 허용하지 않고 순서가 없는 데이터 집합입니다.

-.add(element): 세트에 새로운 요소를 추가합니다.

-.update(iterable): 여러 개의 요소를 한 번에 추가합니다.

-.remove(element): 요소를 삭제하며, 요소가 없으면 **KeyError**가 발생합니다.

-.discard(element): 요소를 삭제하되, 요소가 없어도 에러를 발생시키지 않습니다.

-.pop(): 세트에서 임의의 요소를 제거하고 반환합니다.

-집합 연산: .difference()(차집합), .intersection()(교집합), .union()(합집합), .issubset()(부분집합 여부) 등을 지원합니다.

## defaultdict (collections 모듈)
존재하지 않는 키를 참조할 때 에러를 내지 않고, 설정한 기본값으로 자동 초기화해주는 객체입니다.

효율성: 키가 있는지 매번 확인(if key not in dict)할 필요가 없어 코드가 간결해집니다.

활용 예시: defaultdict(int)는 0으로, defaultdict(list)는 빈 리스트로 자동 시작합니다.

## Hash Table과 Hashable
파이썬의 set과 dict는 내부적으로 해시 테이블을 사용하여 $O(1)$의 탐색 속도를 유지합니다.

-hash() 함수: 객체의 고유한 정수 값을 생성하며, 해시 테이블의 인덱스로 활용됩니다.

-Hashable: int, str, tuple처럼 값이 변하지 않는(Immutable) 객체만 해시가 가능합니다.

-Unhashable: list, dict, set처럼 값이 변할 수 있는(Mutable) 객체는 해시가 불가능하여 set의 요소나 dict의 키로 쓸 수 없습니다.
