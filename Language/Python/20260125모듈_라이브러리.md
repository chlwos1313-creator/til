# 라이브러리/모듈

정의 : 한 파일로 묶인 변수와 함수의 모음
ex) 파이썬 파일(.py)가 하나의 모듈

- math 내장 모듈
    - 파이썬이 미리 작성해 둔 수학 관련 변수와 함수가 작성된 모듈

모듈을 가져오는 법 

importh 문 (권)

```python
import math

print(math.pi) # 3.14..
print(math.sqrt(4)) # 2
```

from 절 사용

```python
from math import pi, sqrt

print(pi)
print(sqrt(4))
```

단점

- 정의된 모듈이 명시적이지 않을 수 있음
- 사용자가 선언한 변수 또는 함수와 겹치게 되어 동작이 이루어 지지 않을 수 있음

as 키워드

```python
from math import sqrt
from my_math import sqrt as my_sqrt
#두 개 이상의 모듈에서 동일한 이름의 변수, 함수 클래스 등을 가져올 때 발생하는 이름 충돌 해결

import pandas as pd
df = pd.DataFrame()

```

# 파이썬 표준 라이브러리

모듈 > 패키지 > 라이브러리

패키지 : 연관된 모듈들을 하나의 디렉토리에 모아 놓은 것

ex) my_package 패키지의 math 모듈의 my_math함수꾸러미

```python
from my_package.math import my_math
from my_package.statistics import tools
```

PIP

외부 패키지들을 설치하도록 도와주는 파이썬의 패키지 관리 시스템

```python
$ pip install Somepackage
$ pip install Somepackage==1.0.5 #특정버전
$ pip uninstall Somepackage
```
